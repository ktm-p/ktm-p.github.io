Build Your Own World

Build Your Own World (BYOW) is a 2D game made in Java where the user tries to collect a key and escape from the maze.

The game features a pseudo-random world generation system made using Prim's Algorithm. It also has a playable character which responds to user input, a lighting system created with Breadth-First Search, limited vision that prevents the user from seeing past walls, and enemies with pathfinding AI implemented using A*-search. Furthermore, there is a save feature where the user can return to the latest save state before they quit.

CS61CPU

CS61CPU is a project where I designed a CPU using Logism.

For the CPU, I implemented the Arithmetic Logic Unit (ALU), Register File (RegFile), Immediate Generator (ImmGen), and implemented the datapaths.

The CPU is capable of performing an assortment of instructions, such as I-type instructions (such as addi, ori, etc.), R-type instructions (add, or, slt, etc.). CS61kaChow

CS61kaChow is a project, written in C, that is focused on implementing optimizations to better the runtime of an algorithm.

The base algorithm I implemented is a naive 2D convolution algorithm which is used to apply different effects (such as blurring, sharpening, etc.) onto videos.

From there, in order to reduce the runtime needed, I applied different techniques such as changing the algorithm so that it dealt with pointers directly via pointer arithmetics and reorganizing the order in which the steps were executed to optimize cache. Furthermore, I used registers to store commonly-used variables, which helped decrease the runtime of the algorithm.

The project also used OpenMP to introduce parallel programming, significantly reducing the runtime even further. Also, by using SIMD instructions to vectorize operations and loop unrolling, the runtime was further reduced. For the CPU, I implemented the Arithmetic Logic Unit (ALU), Register File (RegFile), Immediate Generator (ImmGen), and implemented the datapaths. Overall, the optimizations led to approximately a x50 speedup.

Since this is a class project, please contact me in order to view the source code.

Exploiting Memory Safety Vulnerabilities

Within this project, I looked at a variety of functions written in C and tried to find security exploits in them. Over the course of the project, I had to circumvent memory safety mitigation tactics such as stack canaries, Address Space Layout Randomization (ASLR), and a combination of the two.

Vulnerabilities involved in this project include buffer overflow attacks, integer conversion vulnerabilities, off-by-one vulnerabilities, Time-of-Check to Time-of-Use (TOCTOU), and format string vulnerabilities. I also had to implement a Ret2Ret attack to bypass the ASLR and stack canaries of some functions.

The project involved heavy use of tools such as gdb and being able to analyze memory in order to figure out the potential exploit and also to debug.

Since this is a class project, please contact me in order to view the source code.

A Secure File Sharing System

This is a project which is implemented in Golang whose goal is to, using cryptographic library functions, implement a secure file sharing system.

The system has functionalities such as initializing (unique) users and loading their info on multiple devices. The users can also create files within their namespace, retrieve the specified file they own, and also share files with other users so that they can edit it. The users also have the option to revoke a invitee's access rights.

The project is heavily design-oriented, as I had to design the system, and determine possible vulnerabilities that adversaries can exploit to gain access or tamper with both user and file data.
